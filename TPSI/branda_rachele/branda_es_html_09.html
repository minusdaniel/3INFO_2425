localhost

In computer networking, localhost is a hostname that refers to the current computer in use.

It is used to access network services that are running on the host through the loopback network interface. Using the loopback interface bypasses any local network interface hardware.

Loopback

The loopback mechanism can be used to run a network service on a host without requiring a physical network interface, or without making the service accessible from the networks to which the computer may be connected. For example, a locally installed website can be accessed from a web browser via the URL http://localhost to view its home page. The name localhost normally resolves to the loopback address IPv4 127.0.0.1, and to the loopback address IPv6 ::1.

Name Resolution

Network standardsIPv4reserve the entire block of addresses 127.0.0.0/8 (over 16 million addresses) for loopback purposes. This means that any packet sent to one of those addresses is returned. The address 127.0.0.1 is the standard address for IPv4 loopback traffic; the rest are not supported by all operating systems. However, they can be used to configure multiple server applications on the host, all listening on the same port number. The IPv6 standard assigns only a single address for loopback: ::1.

127.0.0.1 localhost 8 su 10
::1 localhost 8 su 10

The name can also be resolved by Domain Name System (DNS), but queries for this name should be resolved locally and should not be forwarded to remote name servers.
In addition to mapping localhost to loopback addresses (127.0.0.1 and ::1), localhost can also be mapped to other IPv4 (loopback) addresses and it is also possible to assign other names to any loopback address. Mapping localhost to addresses other than those designated in the loopback address range in the hosts file or in DNS is not guaranteed to have the desired effect, as applications may map the name internally.

In the Domain Name System, the name localhost is reserved as a top-level domain name, originally set aside to avoid confusion with the host name used for loopback purposes. IETF standards prohibit domain name registrars from assigning the name localhost.

IETF Standards

The name localhost is reserved for loopback purposes by RFC 6761 (Special Use Domain Names) , which reached Proposed Standard maturity in February 2013. The standard establishes a series of special considerations governing the use of the name in the Domain Name System:
An IPv4 or IPv6 address query for the name localhost must always resolve to the respective loopback address, which is specified in a separate standard.

Applications can resolve the name to a loopback address on their own, or pass it toclass="meter" local name resolution mechanisms. When a name resolver receives an address (A or AAAA) query for localhost, it should return the appropriate loopback addresses and negative responses for any other type of record requested.
Queries for localhost should not be sent to cached name servers. To avoid burdening the root servers of the Domain Name System with traffic, cached name servers should never request name server records for localhost or forward resolution to authoritative name servers. DNS registrars are excluded from delegating domain names in the top-level domain localhost. When authoritative name servers receive queries for localhost despite the aforementioned provisions, they must resolve them appropriately. Loopback addresses IPv4 are reserved within the IPv4 address space by the IETF "Special Use IPv4 Address Standard" (RFC 5735). The reservation can be traced back to the November 1986 standard "Assigned Numbers" (RFC 990).

Packet Processing

The processing of any packet sent to a loopback address is implemented at the link layer of the TCP/IP stack. Such packets are never passed to any network interface controller (NIC) or hardware device driver and should not appear outside of a computer system, nor be routed by any router. This allows for software testing and local services in the absence of hardware network interfaces.
Loopback packets are distinguished from any other packets traversing the TCP/IP stack only by the special IP address to which they were directed. Therefore, services that receive them respond according to the specified destination. For example, an HTTP service might route packets directed to 127.0.0.99:80 and 127.0.0.100:80 to different web servers, or to a single server that returns different web pages. To simplify such testing, the hosts file can be configured to provide appropriate names for each address.

Packets received on a non-loopback interface with a loopback source or destination address must be discarded. Such packets are sometimes referred to as Martian packets. As with any other bogus packet, they can be harmful, and any problems they might cause can be avoided by applying bogon filtering.

Special Cases

Versions of the MySQL database differentiate between using the hostname localhost and using the addresses 127.0.0.1 and ::1. When using localhost as a destination in a client connector interface of an application, the MySQL application programming interface connects to the database using a Unix domain socket, while a TCP connection through the loopback interface requires the direct use of the explicit address.
A notable exception to the use of addresses 127.0.0.0/8 is their use in error detection of Multiprotocol Label Switching (MPLS) traceroute, where their non-routable property provides a convenient means to avoid delivering faulty packets to end users.

127.0.0.1 8 su 10
::1 6.5 su 10
127.0.0.0/8 9.5 su 10
MPLS 4.5 su 10
TCP 7.5 su 10
